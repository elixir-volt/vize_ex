## Key Code — Vapor IR Type

```elixir
@type ir_result :: %{
  templates: [String.t()],      # static HTML fragments (holes replaced with spaces)
  components: [String.t()],     # component names referenced
  directives: [String.t()],     # directive names referenced
  block: map()                  # root block: %{operations: [...], effects: [...], returns: [...]}
}
```

## Vapor IR Output — All Vue Features

### 1. Static Text: `<div>hello</div>`
```elixir
%{
  block: %{returns: [0], operations: [], effects: []},
  components: [],
  templates: ["<div>hello</div>"],
  directives: []
}
```
Static content stays entirely in `templates`. No operations or effects needed.

## Architecture

- **`Vize`** — public Elixir API with `vapor_ir/1`, `compile_vapor/2`, `compile_sfc/2`, etc.
- **`Vize.Native`** — Rustler NIF bridge (delegates to Rust crate `vize_ex_nif`)
- **Rust NIF** — calls Vue's Vapor compiler, converts its IR to Elixir-native maps

### IR Structure Summary

| Top-level key | Type | Purpose |
|---|---|---|
| `templates` | `[String.t()]` | Static HTML fragments with space placeholders for dynamic holes |
| `components` | `[String.t()]` | Component names used |
| `directives` | `[String.t()]` | Directive names used |
| `block` | map | Root block: `{operations, effects, returns}` |

### Operation Kinds
| Kind | Vue Feature | Location |
|---|---|---|
| `:set_text` | `{{ expr }}` | effects |
| `:set_prop` | `:attr="expr"` | effects |
| `:set_html` | `v-html` | effects |
| `:set_dynamic_props` | `v-bind="obj"` | effects |
| `:set_event` | `@event="handler"` | operations |
| `:if_node` | `v-if`/`v-else-if`/`v-else` | operations |
| `:for_node` | `v-for` | operations |
| `:create_component` | `<Component />` | operations |
| `:directive` | `v-show`, `v-model` | operations |

### Key Patterns
- **operations** = structural or one-time setup (events, conditions, loops, components, directives)
- **effects** = reactive updates that re-run when dependencies change (text, props, html)
- **returns** = indices into `templates` array for root element references
- Nested blocks (`:if_node.positive`, `:for_node.render`) have their own `{operations, effects, returns}`
